addObject:withError:

Inserts a given object at the end of the array.

- (BOOL)addObject:(id)anObject withError:(NSError **)errorPtr

Parameters

anObject

The object to add to the end of the array’s content.

             This value may be nil.
errorPtr

A pointer to an NSError object or NULL. You do not need to

             create an `NSError` object.

Return Value

YES if the object was inserted, NO otherwise.

Discussion

Inserts a given object at the end of the array.

If anObject is nil, no NSInvalidArgumentException is raised, but errorPtr will contain an NSError object.

insertObject:atIndex:withError:

Inserts a given object into the array’s contents at a given index.

- (BOOL)insertObject:(id)anObject atIndex:(NSUInteger)index withError:(NSError **)errorPtr

Parameters

anObject

The object to add to the array’s content. This value may be

             nil, but then the object will not be inserted in the array.
index

The index in the array at which to insert anObject. This value may

          be greater than the count of elements in the array, but then the 
          object will not be inserted in the array.
errorPtr

A pointer to an NSError object or NULL. You do not need to

             create an `NSError` object.

Return Value

YES if the object was inserted, NO otherwise.

Discussion

Inserts a given object into the array’s contents at a given index.

If index is greater than the number of elements in the array, no NSRangeException is raised, but errorPtr will contain an NSError object.

If anObject is nil, no NSInvalidArgumentException is raised, but errorPtr will contain an NSError object.

removeLastObjectWithError:

Removes the object with the highest-valued index in the array.

- (BOOL)removeLastObjectWithError:(NSError **)errorPtr

Parameters

errorPtr

A pointer to an NSError object or NULL. You do not need to

             create an `NSError` object.

Return Value

YES if the object was removed, NO otherwise.

Discussion

Removes the object with the highest-valued index in the array.

If there are no objects in the array, no NSRangeException is raised, but errorPtr will contain an NSError object.

removeObjectAtIndex:withError:

Removes the object at index.

- (BOOL)removeObjectAtIndex:(NSUInteger)index withError:(NSError **)errorPtr

Parameters

index

The index from which to remove the object in the array. The value

          may exceed the bounds of the array, but then the object will not
          be removed from the array.
errorPtr

A pointer to an NSError object or NULL. You do not need to

             create an `NSError` object.

Return Value

YES if the object was removed, NO otherwise.

Discussion

Removes the object at index.

If index is beyond the end of the array, no NSRangeException is raised, but errorPtr will contain an NSError object.

replaceObjectAtIndex:withObject:error:

Replaces the object at index with anObject.

- (BOOL)replaceObjectAtIndex:(NSUInteger)index withObject:(id)anObject error:(NSError **)errorPtr

Parameters

index

The index of the object to be replaced. This value may exceed the

          bounds of the array, but then the object will not be replaced.
anObject

The object with which to replace the object at index index in

          the array. This value may be nil, but then the object will not be
          replaced.
errorPtr

A pointer to an NSError object or NULL. You do not need to

             create an `NSError` object.

Return Value

YES if the object was replaced, NO otherwise.

Discussion

Replaces the object at index with anObject.

If index is greater than the number of elements in the array, no NSRangeException is raised, but errorPtr will contain an NSError object.

If anObject is nil, no NSInvalidArgumentException is raised, but errorPtr will contain an NSError object.
