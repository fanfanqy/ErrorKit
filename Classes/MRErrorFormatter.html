Overview

MRErrorFormatter creates localized string representations of NSError objects. There are also methods that aid in error presentation.

Adds methods for stringizing kGADErrorDomain error codes.

Warning: This extension requires the Admob library. Add a GADBannerView.h import to the header prefix of the project.

Adds methods for stringizing kCLErrorDomain error codes.

Warning: This extension requires the Core Location framework. Add a CoreLocation/CoreLocation.h import to the header prefix of the project.

Adds methods for stringizing Parse error codes.

Warning: This extension requires the Parse framework. Add a Parse/Parse.h import to the header prefix of the project.

Adds methods for stringizing TKErrorDomain error codes.

Warning: This extension requires the TransitionKit library. Add a TransitionKit.h import to the header prefix of the project.

Adds helper methods for displaying validation errors.

Warning: This extension requires the Core Data framework. Add a CoreData/CoreData.h import to the header prefix of the project.

Adds methods for stringizing ADErrorDomain error codes.

Warning: This extension requires the iAD framework.

Adds methods for stringizing MKErrorDomain error codes.

Warning: This extension requires the Store Kit framework. Add a MapKitKit/MapKit.h import to the header prefix of the project.

Adds methods for stringizing MFMailComposeErrorDomain error codes.

Warning: This extension requires the MessageUI framework.

Adds methods for stringizing SKErrorDomain error codes.

Warning: This extension requires the Store Kit framework. Add a StoreKit/StoreKit.h import to the header prefix of the project.

Adds methods for stringizing ACErrorDomain error codes.

Warning: This extension requires the Accounts framework. Add a Accounts/Accounts.h import to the header prefix of the project.

Adds methods for stringizing AVFoundationErrorDomain error codes.

Warning: This extension requires the Store Kit framework. Add a AVFoundation/AVFoundation.h import to the header prefix of the project.

Adds methods for stringizing FacebookSDKDomain error codes.

Warning: This extension requires the Facebook SDK library. Add a FacebookSDK.h import to the header prefix of the project.

Adds methods for stringizing JSON patterns.

Adds methods for stringizing VeriJSONErrorDomain error codes.

Warning: This extension requires the VeriJSON library. Add a VeriJSON.h import to the header prefix of the project.

Adds a method that returns a localized message given an exception name.

Adds a method for stringizing HTTP status codes.

Adds methods for stringizing the most common result codes returned by Keychain Services.

Warning: This extension requires the Security framework. Add a Security/Security.h import to the header prefix of the project.

Adds methods for stringizing error codes.

Warning: The strings supplied by these methods are not suitable for using in production environments.

stringsForFoundationErrorCode:

Returns an array that may contain one or more of the following strings for the given error code: localized description, localized failure reason and localized recovery suggestion. The returned strings are gotten from FoundationErrors string table and do not contain any format arguments.

+ (NSArray *)stringsForFoundationErrorCode:(NSInteger)errorCode

Parameters

errorCode

One of the NSError codes defined in FoundationErrors.h.

Return Value

An array with the localized strings or nil if there weren’t any matches for the given errorCode (e.g. NSExecutableArchitectureMismatchError, NSExecutableRuntimeMismatchError, NSExecutableLoadError and NSExecutableLinkError).

Discussion

Returns an array that may contain one or more of the following strings for the given error code: localized description, localized failure reason and localized recovery suggestion. The returned strings are gotten from FoundationErrors string table and do not contain any format arguments.
